Documentation

Process

We originally planned the game to be very similar to Tank Trouble. We wanted a local multiplayer game that would be action-packed and fun. Using the template from Circle Blast, we took the design and code for the player and bullets and implemented into our game. Once we managed to get the players working, we then went about making a randomly generated map with walls. The map generation is done with a grid system, where walls are placed randomly on the grid. The collisions were time-consuming to implement as we tried many different methods that didn't work. Eventually we got bullet collision working well, but the tank to wall collision is still buggy. We decided to only have 2 game states and that it would just keep going back to the main game scene instead of a game over screen. The music wasn't very cooperative as it wasn't preloading so it took me a while to finally get the right parameters in Howler for it to play at the start. The bullets changing colors over time is a significant touch to the game. The effect is appealing and easy for the players to understand. It was also very annoying to concatenate a bunch of hex values and strings.

Problems

Sometimes the tanks spawn encased in walls and they can't get to each other. The only solution we could think of was some sort of pathfinding check. Implementing this would have been difficult and time-consuming though so we decided against it. Instead the player can just shoot themselves to advance to a different level.

The tank to wall collision can sometimes be a little buggy and you can get stuck on walls temporarily.

There may or may not be an edge case where the tanks spawn outside the map or on top of each other. After trying to fix it, I couldn't reproduce the problem so I will just pretend it is solved.

We originally wanted to do a single player mode too, but that quickly got ditched.

Something we definitely could have done was adding power-ups but we start running short on time and we felt that we had spent enough time working on this project.
